README: ### 1. io_uring: Designed for Modern NVMe Storage

**The Problem:** Modern NVMe SSDs can handle 64K+ parallel operations, but traditional blocking I/O creates bottlenecks. **io_uring Solution:** Submit thousands of I/O operations without blocking, saturating NVMe hardware. **Real-world impact:** - rsync: ~420 MB/s (bottlenecked) - arsync: ~850 MB/s (2x faster - saturating NVMe queue depth)

‚öîÔ∏è 2. Security: TOCTOU-Free Metadata Operations

README: ### 2. Security: TOCTOU-Free Metadata Operations

**The Problem:** rsync uses 1980s path-based syscalls vulnerable to race conditions: - CVE-2024-12747: Symlink race condition (Dec 2024, actively exploited) - CVE-2007-4476, CVE-2004-0452: Privilege escalation attacks **arsync Solution:** File descriptor-based operations (fchmod, fchown, fgetxattr, fsetxattr) - Impossible to exploit - FDs are bound to inodes, not paths - Follows MITRE/NIST secure coding guidelines
